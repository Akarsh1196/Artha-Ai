1. Product Vision & Goals
ArthaAI is a resume-grade, fully deployable AI-powered personal finance platform built specifically for Indian users. It combines robust financial logic, real Indian tax laws, Gemini-powered conversational AI, and machine learning predictions — packaged in a professional fintech-grade UI.

Target User
	•	Indian salaried professionals aged 22–45
	•	Users tracking monthly expenses, SIPs, loans, and tax-saving goals
	•	People who want AI guidance without a human financial advisor

What Makes It Resume-Worthy
	•	Full-stack architecture spanning 3 separate services (React, Spring Boot, FastAPI)
	•	Real Indian financial context — 80C, 80D, NPS, SIP compounding, EPF
	•	Gemini LLM integration with structured prompt engineering and context injection
	•	OAuth2 Google login + JWT dual-token auth (access + refresh tokens)
	•	ML microservice for spending anomaly detection and savings forecasting
	•	Live deployed product with a public URL — not just a localhost demo

3. System Architecture
3.1 High-Level Data Flow
The system is split into three independently deployable services connected via REST APIs:

Layer
Responsibility
Communicates With
React (Vercel)
UI rendering, state management, Gemini streaming chat
Spring Boot via Axios/React Query
Spring Boot (Railway)
Business logic, auth, scoring engine, DB access
PostgreSQL, Python ML service, Gemini API (server-side)
Python FastAPI (Render)
ML model inference: anomaly detection, forecasting
Called by Spring Boot via WebClient
PostgreSQL (Neon)
Single source of truth for all user data
Spring Boot via JDBC / Hibernate
Gemini API
LLM for chatbot and AI-generated advice text
Spring Boot (server-side proxy) or React (direct stream)

3.2 Authentication Flow
	•	User lands on /login — two options: email+password OR Continue with Google
	•	Email/password: POST /api/auth/signup → Spring Security hashes password with BCrypt (strength 12) → stores in users table → returns access token (15 min) + refresh token (7 days, HttpOnly cookie)
	•	Google OAuth2: Spring Security OAuth2 Client → Google consent screen → callback /login/oauth2/code/google → Spring creates/updates user in DB → same JWT pair issued
	•	Every protected API request: Axios interceptor reads access token from memory → sends as Authorization: Bearer header → Spring JwtAuthFilter validates → injects UserPrincipal into SecurityContext
	•	Token refresh: Axios response interceptor detects 401 → calls POST /api/auth/refresh with HttpOnly cookie → Spring returns new access token → original request retried

3.3 Gemini Chat Proxy Architecture
Two modes depending on context size:
	•	Short chats (< 500 tokens context): React calls Gemini SDK directly with streaming — instantaneous UI feel
	•	Full advisory queries: React calls POST /api/ai/chat → Spring Boot builds enriched prompt with full user profile → calls Gemini API server-side → streams SSE back to React
	•	Server-side proxy hides Gemini API key, enforces rate limits, logs usage, saves history

6. Financial Health Score Engine
The scoring engine is the core differentiator of ArthaAI. It runs server-side in Spring Boot and produces a 0–100 score with sub-scores and an AI-generated commentary.

6.1 Scoring Parameters & Weights
Parameter
Weight
Formula
Scoring Logic
Savings Ratio
30%
Monthly Savings / Monthly Income
≥40%=100, 25–40%=75, 10–25%=50, <10%=20
Debt-to-Income (DTI)
25%
Total Monthly EMIs / Monthly Income
≤15%=100, 15–30%=75, 30–50%=50, >50%=20
Emergency Fund
20%
Liquid savings ÷ Monthly expenses
≥6 months=100, 3–6=70, 1–3=40, <1=10
Investment Consistency
15%
Active SIPs or monthly investments
Regular SIPs=100, occasional=60, none=0
Insurance Coverage
10%
Has health insurance declared
Yes=100, No=0

6.2 Final Score Calculation
Final Score = (SavingsScore × 0.30) + (DTIScore × 0.25) + (EmergencyScore × 0.20) + (InvestmentScore × 0.15) + (InsuranceScore × 0.10)

6.3 Gemini Commentary Prompt
After scoring, Spring Boot sends this prompt to Gemini 1.5 Flash:

You are a concise Indian financial advisor. User score: {score}/100. Sub-scores: savings={s}, debt={d}, emergency={e}, investment={i}. Write exactly 2 sentences: first, acknowledge their biggest strength. Second, name ONE specific improvement action in plain Hindi-friendly English. No markdown, no lists.


7. Python ML Microservice
7.1 Anomaly Detection — IsolationForest
	•	Input: Last 90 days of expense records as JSON array from Spring Boot
	•	Features used: amount, day_of_week, category_encoded, is_weekend
	•	Model: sklearn.ensemble.IsolationForest with contamination=0.05 (expects ~5% anomalies)
	•	Output: List of expense IDs flagged as anomalous with anomaly_score
	•	Spring Boot receives this and stores flags in expenses.is_anomaly column, triggers in-app notification

7.2 Savings Forecast — Linear Regression
	•	Input: Last 6–12 months of (month_index, monthly_savings) pairs
	•	Model: sklearn.linear_model.LinearRegression — simple and explainable
	•	Output: Next 6 months of predicted savings as array, plus confidence_low and confidence_high (±1 std)
	•	Displayed in dashboard as a projected savings line chart with shaded confidence band

15-minute expiry, signed with HS512, stored in memory (not localStorage)
JWT refresh token
7-day expiry, stored as HttpOnly + Secure + SameSite=Strict cookie
Google OAuth
Spring Security OAuth2 Client handles PKCE flow — no secret exposed to browser
CORS
Allowlist only Vercel frontend domain in production
Rate limiting
Bucket4j: 20 AI requests/user/hour, 100 API requests/user/minute
SQL injection
All DB access via Hibernate JPA — zero raw JDBC queries
Input validation
@Valid on all request DTOs — validated at controller layer
API key security
Gemini API key in Railway env vars only — never in frontend bundle
HTTPS
Enforced by Vercel (FE) and Railway (BE) — HTTP redirected to HTTPS
Secrets management
.env files gitignored — .env.example committed as template
